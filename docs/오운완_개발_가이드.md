## 1. 제품 포지셔닝 & 리서치 인사이트

국내외 27개 앱(바디캘린더·번핏·플랜핏·눈바딩·체중메이트·Strong·Hevy·JEFIT·Fitbod·Strava 등)을 조사한 핵심:

- **바디캘린더는 "다이어트 앱"이 아니라 "웨이트 트레이닝 로깅 앱"**. 이름과 달리 눈바디(진행사진)·식단 기능이 없다.

- 유저 3대 불만: ①광고 ②기능 과다·초기 복잡·렉 ③구독가. 반복 칭찬: 스마트워치 연동, 캘린더 직관성/드래그복사, 진척 시각화(1RM/눈바디).
- **승부수(포지셔닝)**: "바벨만 기억하는 게 아니라 내 몸까지 기억하는, 가장 빠르고 예쁜 잔디 캘린더 운동 기록 앱." 솔로 빌더는 기능으로 못 이기니 ①초고속 입력 ②캘린더/연속기록 습관 루프에 집중.

## 2. 기술 스택 & 아키텍처

- **Next.js 16** (App Router, **정적 익스포트** `output: "export"`) + **React 19** + **TypeScript** + **Tailwind v4**(CSS-first, `@theme`)
- **Dexie(IndexedDB)** — 로컬퍼스트 저장소(오프라인 즉시 기록)
- **Supabase** — 인증(이메일 매직링크) + Postgres + 기기 간 동기화 (선택 — 없으면 게스트 로컬 모드로 완전 동작)
- **@tanstack/react-query** — 조회/뮤테이션 캐시
- 차트·아이콘(lucide-react) 외 무거운 의존성 없음. 차트는 자체 SVG 구현.
- **PWA**: manifest + 서비스워커(오프라인 셸, 프로덕션에서만 등록) + 설치 아이콘. Pretendard 폰트(CDN).

### 아키텍처 핵심: 게스트 / 로그인 이중 구조
- **게스트 모드**: `ownerId = 'local'`, Dexie에만 저장. Supabase 미설정이어도 모든 기능 동작 → 프리뷰·데모 무설정 가능.
- **로그인 모드**: `ownerId = user.id`. Dexie(로컬 즉시) + Supabase 동기화. 로그인 시 로컬('local') 데이터를 계정으로 1회 이관(`migrateLocalDataTo`).
- **모든 화면은 클라이언트 컴포넌트**. 정적 익스포트라 서버 의존 없음.

### 동기화 (sync.ts)
- 레코드를 **JSONB 문서**로 Supabase에 저장(테이블: profiles/exercises/routines/sessions/body_metrics, 각 `id/owner_id/data/updated_at/deleted_at` + RLS로 본인 데이터만).
- **로컬퍼스트**: 모든 레코드에 `updatedAt`(ISO) + `_dirty`(로컬 push 필요) + `deletedAt`(소프트삭제).
- `fullSync()` = push(dirty→upsert) → pull(updated_at > 커서). **Last-Write-Wins**: 원격 `updated_at`을 `new Date().toISOString()`로 정규화 후 `Date.parse`(epoch ms)로 비교(PostgREST `+00:00` vs `Z` 포맷 불일치 방지).
- **pull로 원격 변경이 들어오면 `onRemoteChange()` → React Query invalidate → UI 즉시 갱신** (모바일↔PC 최신화).
- 트리거: 로그인 직후, 창 포커스/온라인 복귀/visibility, 뮤테이션 후 디바운스(`scheduleSync`), (선택)realtime.
- **빌트인 운동 151종은 동기화 대상 아님**(모두 공통 → 로컬 시드). 커스텀 운동만 동기화.

## 3. 데이터 모델 (`src/lib/types.ts`)

- `Profile` — unit(kg/lb), theme, weekStartsMonday, onboardedAt
- `Exercise` — id(빌트인=slug, 커스텀=uuid), nameKo/nameEn, bodyPart(8분류), primaryMuscle, equipment, category(strength/cardio/bodyweight/stretching), defaultRestSeconds, isBuiltIn …
- `Routine` — name, emoji, folder, `exercises: RoutineExerciseRef[]`({exerciseId, targetSets})
- `WorkoutSession` — date(YYYY-MM-DD), title, sessionIndexOfDay(하루 다중), routineId, **startedAt/endedAt**(타이머), bodyweight, `exercises: SessionExercise[]`, 파생캐시(bodyParts/totalVolume/totalSets)
- `SessionExercise` — exerciseId, orderIndex, **note(메모 30자)**, **trackingMode**('weight_reps'|'reps'|'time'), **restSeconds(운동별 휴식 override)**, `sets: WorkoutSet[]`
- `WorkoutSet` — setType('working'|'warmup'|'drop'|'failure'), weight(kg 저장), reps, **durationSec(시간 방식)**, isCompleted, completedAt(세트 체크시각)
- `BodyMetric` — date, weight, bodyFatPct, muscleMass
- 빌트인 운동: `src/data/exercises.json`(151종) → `src/data/exercises.ts`에서 `id=slug`로 매핑(그래서 **빌트인 exerciseId == slug**).

## 5. 로깅 화면 규칙 (중요)

### 타이머 (시작/종료) — R2에서 도입, R5에서 버그수정
- 진입 즉시 시간이 흐르지 않음. 상태 3가지:
  - **시작 전**(`startedAt` null): 헤더에 "시작 전" + `[운동 시작]` 버튼(운동 0개면 비활성).
  - **진행 중**(`startedAt` 있고 `endedAt` null): "⏱ mm:ss"(브랜드색) 흐름 + `[운동 완료]`. `startedAt` 기준 `now-start`.
  - **완료**(`startedAt`·`endedAt` 둘 다): "⏱ mm:ss · 완료" **고정**(`endedAt-startedAt`) + `[운동 이어서]`.
- **첫 세트 완료 시 타이머 자동 시작**(startedAt 자동 세팅).
- `[운동 완료]` → `endedAt=now`, 저장, 홈 이동. 완료된 세트 체크시각으로 startedAt 보정.
- **[운동 이어서]** → 확인 팝업 "완료 시점 시간에서 이어서 다시 시작할까요?" → "이어서 하기": `startedAt = now - 기존경과`, `endedAt=null`로 되돌려 **이전 시간에서 이어서 흐름**. "취소": 그대로 고정.
- (R5 수정 전 버그: 경과시간이 `endedAt`을 무시하고 `now-startedAt`으로만 계산돼 완료된 과거 세션 재진입 시 초가 계속 흐르던 문제 → `endedAt`이 있으면 고정하도록 수정.)

### 기록 방식 (trackingMode) — 운동 카드 ⋮ 메뉴
- **중량 + 횟수**(기본): [KG][횟수] 입력. 볼륨·PR 집계됨.
- **횟수만**(맨몸): [횟수]만.
- **시간만**(유산소·스트레칭): **[분][초] 2개 입력**(내부 저장 `durationSec = 분×60+초`, 초는 0~59). 볼륨 미집계.
- **자동 기본값**: `defaultModeFor(category)` — cardio·stretching → 시간, bodyweight → 횟수, 그 외 → 중량+횟수.
- 지난기록 불러오기·복사/붙여넣기에 기록방식·휴식·메모·시간값 모두 반영.

### 휴식 시간 — 운동 카드 우측 "⏱ mm:ss" 칩
- 탭 → 프리셋 시트(끄기/0:30/0:45/1:00/1:30/2:00/2:30/3:00/4:00/5:00). 운동별 override(`restSeconds`), 미설정 시 운동 기본값.
- 세트 완료 시 이 값으로 하단 휴식 타이머 자동 시작(0이면 끔).

### 세트 유형 & 세트 삭제 — 세트 번호(1,2,3…) 탭 → "세트 설정" 시트 (R5)
- **본세트**(볼륨 집계) / **웜업 W**(볼륨 집계 제외) / **드랍셋 D** / **실패셋 F**(기록용 태그) — 설명과 함께 선택.
- **"이 세트 삭제"** 버튼으로 개별 세트 삭제(모바일 포함). (이전의 발견 불가능한 우클릭 삭제·번호탭 순환 제거)
- 볼륨 계산은 `working/drop/failure`만 집계, `warmup` 제외.

### 메모 — 운동 카드 이름 아래 인라인 입력
- 최대 한글 30자. 자세·주의점 기록(예: "견갑 고정, 반동 없이").

### 세트 입력 UX
- 각 값은 −/입력/+ 스테퍼. 중량 ±2.5(kg)/±5(lb), 횟수 ±1, 시간 초 ±15. 지난 기록이 placeholder로 자동 표시.

## 8. 알려진 제약 & 앞으로

- 스마트워치 연동은 PWA 한계로 미지원(알려진 제약).
- **백그라운드 휴식 알림(R10)**: 예약된 오디오+keep-alive로 대부분의 안드로이드 백그라운드에서 종료음이 울리지만, OS가 탭/오디오를 강하게 절전 종료하면 네이티브처럼 100% 보장은 안 됨(웹 플랫폼 한계). 화면 꺼짐/장시간 방치 등 극단 케이스는 미보장. 카운트다운 **진동**은 포그라운드에서만 동작. (더 강한 보장이 필요하면 서버 Web Push 스케줄링이 필요 — 미도입)
- 눈바디(진행사진) 다이어리 = 바디캘린더가 없는 차별점, 향후 should-have 후보.
- 결제·AI 프로그래밍·소셜피드는 의도적 보류.
- **다음 예정**: 추천 4/5분할, (요청 시) 눈바디·리캡·위젯 등.
